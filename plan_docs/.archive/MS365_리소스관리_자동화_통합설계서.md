## 1. 프로젝트 개요

본 설계서는 MS 365 생태계(Excel, SharePoint, Python, Power Virtual Agents)를 활용하여 조직의 인력 리소스를 효율적으로 관리하고, AI 기반 질의응답을 통해 리소스 배치 의사결정을 지원하는 시스템을 설명합니다.

기존 Power Automate 기반 설계에서 Python 스크립트 기반 동기화로 전환하여 운영 안정성과 유지보수 편의성을 높였습니다.

### 1.1 목적 및 범위

| 구분 | 내용 |
|------|------|
| 목적 | 인력 현황 실시간 파악 및 AI 기반 리소스 매칭 자동화 |
| 대상 범위 | 전체 조직원 (약 70명 기준) 리소스 데이터 통합 관리 |
| 기대 효과 | 리소스 배치 의사결정 시간 단축, 데이터 신선도 유지, AI 질의응답 지원 |
| 적용 플랫폼 | MS 365 (Excel Online, SharePoint, Power Virtual Agents) + Python |

### 1.2 전체 아키텍처 (5단계)

| 단계 | 구성 요소 | 역할 |
|------|-----------|------|
| 1단계 | Excel (ResourceTable) | 마스터 데이터 입력 및 관리 |
| 2단계 | SharePoint List | 단일 진실 공급원(Single Source of Truth) |
| 3단계 | Python 스크립트 + 작업 스케줄러 | 데이터 자동 동기화 (Power Automate 대체) |
| 4단계 | Power Virtual Agents (PVA) for Teams | AI 에이전트 자연어 질의 처리 |
| 5단계 | Management Loop | 운영 프로세스 및 리포트 자동화 |

> **✅ Power Automate → Python 전환 배경**
> - Power Automate 플로우에서 Null 값 처리 오류 반복 발생
> - 비개발자가 Power Automate 오류 원인 파악 및 수정이 어려움
> - Python 스크립트는 오류 메시지가 명확하고 수동 재실행이 간단함
> - Windows 작업 스케줄러로 동일한 자동화 주기 구현 가능
> - 추가 라이선스 비용 없이 동일 기능 구현

## 2. 1단계: 마스터 데이터 구조 설계 (Excel)

AI와 Python 스크립트가 읽기 좋은 구조화된 Excel 표(Table)를 설계합니다.

> **파일 설정**
> - 파일명: `ResourceManagement.xlsx`
> - 저장 경로: SharePoint Document Library > 리소스관리 폴더
> - 테이블명: `ResourceTable` (Excel [삽입] → [표] 로 지정)
> - 접근 권한: 운영팀장 이상 편집 권한 / 전체 읽기 권한

### 2.1 컬럼 구성 (ResourceTable)

| # | 컬럼명 | 데이터 타입 | 예시 | 설명 및 입력 규칙 |
|---|--------|------------|------|-----------------|
| 1 | 성명 | 텍스트 | 홍길동 | 직원 전체 이름 (고유 식별자, Python Upsert 기준키) |
| 2 | 팀 | 텍스트 | 개발1팀 | 소속 팀명 |
| 3 | 직급 | 텍스트 | 선임 | 인턴/주임/선임/책임/팀장 등 |
| 4 | 주요기술 | 텍스트 | Java, Spring, AWS | 쉼표(,)+공백으로 구분 필수 |
| 5 | 숙련도 | 숫자(1~5) | 4 | 1=입문 2=초급 3=중급 4=고급 5=전문가 |
| 6 | 현재 프로젝트명 | 텍스트 | A사 ERP 구축 | 없으면 '대기' 입력 |
| 7 | 역할 | 텍스트 | 백엔드 개발 | 현재 프로젝트 내 담당 역할 |
| 8 | 투입률(%) | 숫자(0~100) | 80 | 100이면 풀 투입, 대기 시 0 입력 |
| 9 | 종료 예정일 | 날짜 | 2025-07-31 | YYYY-MM-DD 형식 |
| 10 | 가동 가능일 | 날짜(수식) | =I2+1 | 종료 예정일+1일 자동 계산, 직접 입력 금지 |

> **⚠️ 운영 규칙**
> - 성명 컬럼 공백 금지 — Python 스크립트가 성명을 기준키로 Upsert 처리함
> - 빈 행이 있으면 Python 스크립트가 자동으로 건너뜀 (오류 없이 처리)
> - 주요기술은 반드시 쉼표+공백으로 구분: `'Java, Spring, AWS'`

## 3. 2단계: 데이터 저장소 준비 (SharePoint List)

Excel 마스터 데이터를 받아줄 SharePoint 목록을 구성합니다. Python 스크립트와 PVA AI 에이전트가 이 목록을 참조합니다.

### 3.1 SharePoint 목록 생성 절차

1. SharePoint 사이트 접속 → [새로 만들기] → [목록] 선택
2. [Excel에서] 옵션 선택 후 `ResourceManagement.xlsx` 업로드
3. 컬럼 구조 자동 인식 → 목록명: `ResourceList`
4. 생성 완료 후 아래 표 기준으로 컬럼 타입 검토 및 조정

### 3.2 SharePoint 컬럼 타입 설정

| 컬럼명 | SharePoint 컬럼 타입 | 비고 |
|--------|---------------------|------|
| 성명(Title) | 한 줄 텍스트 | 필수 입력, Python Upsert 기준키 |
| 팀 | 선택 (드롭다운) | 팀 목록 미리 정의하여 오탈자 방지 |
| 직급 | 선택 (드롭다운) | 직급 체계에 맞춰 목록 정의 |
| 주요기술 | 여러 줄 텍스트 | 쉼표 구분 문자열 저장 |
| 숙련도 | 숫자 | 1~5 범위 |
| 현재 프로젝트명 | 한 줄 텍스트 | 없으면 '대기' 기본값 |
| 역할 | 한 줄 텍스트 | 자유 입력 |
| 투입률(%) | 숫자 | 0~100 범위 |
| 종료 예정일 | 날짜 및 시간 | 날짜만 표시로 설정 |
| 가동 가능일 | 날짜 및 시간 | Python 스크립트에서 계산하여 업데이트 |

### 3.3 목록 뷰(View) 설정

| 뷰 이름 | 필터 조건 | 활용 목적 |
|---------|-----------|----------|
| 전체 리소스 | 필터 없음 | 전체 현황 조회 |
| 즉시 가동 가능 | 투입률=0 OR 가동가능일≤오늘 | 배치 가능 인원 빠른 확인 |
| 월별 종료 예정 | 종료예정일=이번달 | 프로젝트 종료 예정자 모니터링 |

#### 4.4.2 sync_resource.py (메인 스크립트)

```python
import msal, requests
from datetime import datetime
from config import *

def get_access_token():
    app = msal.ConfidentialClientApplication(
        CLIENT_ID,
        authority=f'https://login.microsoftonline.com/{TENANT_ID}',
        client_credential=CLIENT_SECRET
    )
    result = app.acquire_token_for_client(
        scopes=['https://graph.microsoft.com/.default']
    )
    if 'access_token' not in result:
        raise Exception(f'토큰 발급 실패: {result.get("error_description")}')
    return result['access_token']

def read_excel_data(token):
    site_resp = requests.get(
        f'https://graph.microsoft.com/v1.0/sites/{EXCEL_SITE_URL.replace("https://","")}',
        headers={'Authorization': f'Bearer {token}'}
    ).json()
    site_id = site_resp['id']
    drives  = requests.get(
        f'https://graph.microsoft.com/v1.0/sites/{site_id}/drives',
        headers={'Authorization': f'Bearer {token}'}
    ).json()
    drive_id = drives['value'][0]['id']
    file_resp = requests.get(
        f'https://graph.microsoft.com/v1.0/drives/{drive_id}/root:{EXCEL_FILE_PATH}',
        headers={'Authorization': f'Bearer {token}'}
    ).json()
    file_id = file_resp['id']
    rows_resp = requests.get(
        f'https://graph.microsoft.com/v1.0/drives/{drive_id}/items/{file_id}'
        f'/workbook/tables/{EXCEL_TABLE}/rows',
        headers={'Authorization': f'Bearer {token}'}
    ).json()
    return rows_resp.get('value', [])

def get_sharepoint_items(token):
    site_resp = requests.get(
        f'https://graph.microsoft.com/v1.0/sites/{SHAREPOINT_SITE_URL.replace("https://","")}',
        headers={'Authorization': f'Bearer {token}'}
    ).json()
    site_id = site_resp['id']
    items_resp = requests.get(
        f'https://graph.microsoft.com/v1.0/sites/{site_id}/lists/{LIST_NAME}/items?expand=fields',
        headers={'Authorization': f'Bearer {token}'}
    ).json()
    result = {}
    for item in items_resp.get('value', []):
        name = item['fields'].get('Title', '')
        if name:
            result[name] = {'id': item['id'], 'fields': item['fields']}
    return result, site_id

def row_to_fields(values):
    # Excel 컬럼 순서: 성명, 팀, 직급, 주요기술, 숙련도,
    #                   현재프로젝트명, 역할, 투입률, 종료예정일, 가동가능일
    return {
        'Title':          str(values[0]).strip(),
        '팀':             str(values[1]).strip(),
        '직급':           str(values[2]).strip(),
        '주요기술':       str(values[3]).strip(),
        '숙련도':         str(values[4]).strip(),
        '현재프로젝트명': str(values[5]).strip(),
        '역할':           str(values[6]).strip(),
        '투입률':         str(values[7]).strip(),
        '종료예정일':     str(values[8]).strip(),
        '가동가능일':     str(values[9]).strip(),
    }

def send_teams_notification(created, updated, errors):
    if not TEAMS_WEBHOOK_URL: return
    msg = {'text':
        f'✅ ResourceTable 동기화 완료\n'
        f'신규:{created}건 / 업데이트:{updated}건 / 오류:{errors}건\n'
        f'실행시간: {datetime.now().strftime("%Y-%m-%d %H:%M")}'
    }
    requests.post(TEAMS_WEBHOOK_URL, json=msg)

def main():
    print(f'[{datetime.now().strftime("%H:%M:%S")}] 동기화 시작...')
    created = updated = errors = 0
    try:
        token = get_access_token()
        print('  ✓ 토큰 발급 성공')
        excel_rows = read_excel_data(token)
        print(f'  ✓ Excel {len(excel_rows)}행 읽기 완료')
        sp_items, site_id = get_sharepoint_items(token)
        print(f'  ✓ SharePoint 기존 {len(sp_items)}건 조회 완료')
        for row in excel_rows:
            values = row.get('values', [[]])[0]
            if not values or not str(values[0]).strip(): continue  # 빈 행 건너뜀
            try:
                fields = row_to_fields(values)
                name   = fields['Title']
                if name in sp_items:
                    requests.patch(
                        f'https://graph.microsoft.com/v1.0/sites/{site_id}'
                        f'/lists/{LIST_NAME}/items/{sp_items[name]["id"]}/fields',
                        headers={'Authorization':f'Bearer {token}','Content-Type':'application/json'},
                        json=fields)
                    updated += 1; print(f'  → 업데이트: {name}')
                else:
                    requests.post(
                        f'https://graph.microsoft.com/v1.0/sites/{site_id}/lists/{LIST_NAME}/items',
                        headers={'Authorization':f'Bearer {token}','Content-Type':'application/json'},
                        json={'fields': fields})
                    created += 1; print(f'  → 신규 생성: {name}')
            except Exception as e:
                errors += 1; print(f'  ✗ 오류({values[0]}): {e}')
        print(f'\n완료! 신규:{created} / 업데이트:{updated} / 오류:{errors}')
        send_teams_notification(created, updated, errors)
    except Exception as e:
        print(f'치명적 오류: {e}')

if __name__ == '__main__': main()
```

## 5. 4단계: AI 에이전트 연동 (Power Virtual Agents for Teams)

SharePoint ResourceList 데이터를 기반으로 자연어 질문에 응답하는 AI 리소스 매니저를 구성합니다.

> **✅ PVA for Teams 무료 제공 조건**
> - 대상 플랜: MS 365 Business Basic / Standard / Premium 전 플랜
> - 무료 사용 조건: Teams 채널 내부 배포 + 표준 커넥터만 사용
> - 무료 세션 한도: 월 2,000 세션 / 테넌트 (70명 내부 운영 시 충분)
> - 유료 전환 조건: 외부 웹사이트 배포, 프리미엄 커넥터, 세션 초과 시
>
> ※ Power Virtual Agents는 2023년 11월 Microsoft Copilot Studio로 리브랜딩. 접속 시 Copilot Studio 화면으로 이동하나 동일한 서비스입니다.

### 5.1 에이전트 기본 설정

| 항목 | 설정값 |
|------|--------|
| 에이전트명 | 리소스 매니저 (Resource Manager) |
| 플랫폼 | Power Virtual Agents (현재명: Microsoft Copilot Studio) |
| 지식 기반 | SharePoint List: ResourceList |
| 배포 채널 | Microsoft Teams (조직 내 배포) |
| 접근 권한 | 운영팀장 이상 (Teams 앱 배포 정책으로 제한) |

### 5.2 시스템 프롬프트 설정

```
너는 블루어드의 리소스 매니저야.
SharePoint 목록 'ResourceList'의 데이터를 실시간으로 참조하여 답변해줘.

답변 시 반드시 아래 규칙을 따라:
1. '가동 가능일'이 오늘 이전이거나 '투입률'이 0%인 사람을 우선 추천해줘.
2. 기술 검색 시 '주요기술' 컬럼의 쉼표 구분 값을 기준으로 매칭해.
3. 결과는 표 형식으로 성명, 팀, 주요기술, 가동 가능일을 포함해서 보여줘.
4. 데이터가 없거나 불확실할 때는 추측하지 말고 '데이터를 확인할 수 없습니다'라고 답해.
```

### 5.3 지식 기반 연결 절차

1. PVA 접속 (`aka.ms/PVAPortal` → Copilot Studio로 자동 이동)
2. 에이전트 선택 → [지식(Knowledge)] 탭 클릭
3. [추가] → [SharePoint] → ResourceList URL 입력
4. 동기화 주기: 매일 자동 갱신
5. 저장 및 게시(Publish) → Teams 배포

### 5.4 예상 질의/응답 시나리오

| 질문 예시 | AI 응답 방식 |
|----------|------------|
| 6월에 Java 개발자 중 투입 가능한 사람이 누구야? | 가동가능일≤6월 + 주요기술 Java 필터링 후 표 출력 |
| 지금 당장 투입 가능한 인원이 몇 명이야? | 투입률=0% 또는 가동가능일≤오늘 인원 카운트 |
