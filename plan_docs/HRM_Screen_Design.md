## 문서 구성 규칙

### 공통 레이아웃

로그인 화면을 제외한 모든 화면은 다음 공통 레이아웃을 따른다.

```
┌─────────────────────────────────────────────────────────────────────┐
│ HRM 자동화 시스템                [역할] [사용자명 ▾]  [알림🔔] [로그아웃] │  ← GNB (상단 고정)
├──────────────┬──────────────────────────────────────────────────────┤
│ 📊 대시보드   │                                                        │
│ 👤 사원 관리  │                                                        │
│ 🔧 기술 관리  │              [ 콘텐츠 영역 ]                             │
│ 💼 직무 유형  │                                                        │
│ 📁 프로젝트   │                                                        │
│ 📋 투입 관리  │                                                        │
│ 🟢 가동 인력  │                                                        │
│ ⭐ 추천       │                                                        │
│ 🤖 AI 질의   │                                                        │
│ 📈 리포트     │                                                        │
│ ⚙️  설정      │                                                        │
└──────────────┴──────────────────────────────────────────────────────┘
  ← LNB (좌측 고정, 권한별 메뉴 노출 제어)
```

### 권한 코드 약어

| 약어 | 역할 (`ROLE_CD`) |
|---|---|
| A | Admin |
| H | HR_MGR |
| P | PM |
| T | TEAM_LEAD |
| E | EXEC |
| V | VIEWER |

### 화면 ID 체계

`SCR-{3자리 순번}` 형식으로 부여한다.

## SCR-001 · 로그인

### 기본 정보

| 항목 | 내용 |
|---|---|
| 화면명 | 로그인 |
| URL | `/login` |
| 접근 권한 | 인증 불필요 (비로그인 상태) |
| 목적 | `SYS_USER_MST` 기반 ID/PW 인증 후 JWT 발급 |
| 관련 API | `POST /api/v1/auth/login` |

### 구성 요소

| 컴포넌트 | 타입 | 설명 | 필수 |
|---|---|---|---|
| 시스템 로고/명칭 | Text | `HRM 자동화 시스템` | — |
| 로그인 ID 입력 | Input (text) | `SYS_USER_MST.USER_LGID` | ✓ |
| 비밀번호 입력 | Input (password) | `SYS_USER_MST.ENCR_PWD` 대조 | ✓ |
| 오류 메시지 | Alert | 인증 실패 시 표시 (로그인 ID 또는 비밀번호 불일치) | — |
| 로그인 버튼 | Button (primary) | `POST /api/v1/auth/login` 호출 | — |

### 동작 정의

| 이벤트 | 동작 |
|---|---|
| 로그인 성공 | Access Token + Refresh Token 발급 → `/dashboard` 이동 |
| 로그인 실패 | 오류 메시지 표시, 입력 필드 초기화 안 함 |
| `USE_YN = FALSE` 계정 | "비활성 계정입니다. 관리자에게 문의하세요." 표시 |
| Enter 키 | 로그인 버튼과 동일하게 동작 |

### 유효성 검사

| 조건 | 메시지 |
|---|---|
| 로그인 ID 미입력 | "로그인 ID를 입력하세요." |
| 비밀번호 미입력 | "비밀번호를 입력하세요." |
| 인증 실패 | "로그인 ID 또는 비밀번호가 올바르지 않습니다." |
| Rate Limit 초과 | "잠시 후 다시 시도하세요." (IP 기준 분당 10회 초과 시) |

## 공통 정책 요약

### 에러 처리

| HTTP 상태 | 처리 방식 |
|---|---|
| 401 Unauthorized | 자동 Refresh Token 갱신 시도 → 실패 시 `/login` 이동 |
| 403 Forbidden | "접근 권한이 없습니다." 토스트 메시지 |
| 404 Not Found | "요청한 데이터를 찾을 수 없습니다." |
| 422 Validation Error | 입력 필드별 오류 메시지 인라인 표시 |
| 500 Server Error | "일시적인 오류가 발생했습니다. 잠시 후 다시 시도하세요." |
