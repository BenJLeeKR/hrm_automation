## 1. 원칙

- **화면 접근 권한**은 화면 설계서(`[DESIGN]HRM_Screen_Design.md`) "화면 목록" 표의 역할 기준을 그대로 따른다.
- **버튼 권한**은 6개 카테고리로 단순화한다: `view`(조회) / `create`(등록) / `update`(수정) / `delete`(삭제) / `excel`(Excel Import·Export) / `admin`(관리자 기능)
- 화면 설계서에 명시되지 않은 세부 항목(예: 프로젝트 목록의 "등록" 버튼 역할, Excel Export의 역할 제한)은 **동일 화면의 수정(update) 권한 그룹과 동일하게 추정**하고, 표에 "(추정)"으로 표기했다. 운영팀 확정 전까지 보수적으로 좁게(적을수록 안전한 방향으로) 설정했다.
- 팀/프로젝트 단위 스코프(예: `TEAM_LEAD`가 "본인팀"만 수정 가능)는 화면·버튼 노출 여부(`PERM_JSON`)로는 표현하지 않는다. `PERM_JSON`은 "이 버튼이 보이는가/눌리는가"만 결정하고, row-level 데이터 스코프는 API 레이어에서 별도 구현한다.
- `SCR-001 로그인`은 비인증 상태 화면이라 권한 매트릭스에서 제외한다.
- `SCR-014 설정(허브)`의 하위 카드 중 "알림 채널/배치 관리/백업 관리"는 화면 설계서에 별도 SCR 상세 정의가 없어 이번 Seed 대상에서 제외한다 (TBD — 별도 화면 설계 확정 후 추가).

## 5. 운영팀 확인 필요 사항 (MVP 초안 → 확정 전 재검토 권장)

1. `employees` Excel Export의 역할 제한 — 화면 설계서 미명시, 현재 Import와 동일(A H)로 보수적으로 설정.
2. `projects` "등록" 버튼 역할 — 화면 설계서 미명시, 현재 "수정"과 동일(A H P)로 추정.
3. `assignments` Excel Export 역할 — TEAM_LEAD 포함 여부 미확정, 현재 제외.
